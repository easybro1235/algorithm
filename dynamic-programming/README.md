# 동적 프로그래밍 (Dynamic Programming)

## 개념
동적 프로그래밍은 큰 문제를 작은 부분 문제로 나누어 해결하고, 결과를 저장하여 재사용하는 기법입니다.

## 핵심 원리
1. **최적 부분 구조** (Optimal Substructure): 큰 문제의 최적해가 작은 문제의 최적해로 구성
2. **중복 부분 문제** (Overlapping Subproblems): 같은 부분 문제가 여러 번 계산됨

## 접근 방법
- Top-down (재귀 + 메모이제이션)
- Bottom-up (반복문 + 테이블)

## 주요 유형
- 피보나치 수열
- 배낭 문제 (Knapsack)
- LCS (Longest Common Subsequence)
- LIS (Longest Increasing Subsequence)
- 편집 거리 (Edit Distance)
- 행렬 곱셈 최적화
